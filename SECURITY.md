# Security Policy

## 🛡️ Security Measures
This project follows industry best practices for Smart Contract security:
1. **Modular Design:** Using battle-tested OpenZeppelin libraries.
2. **Pull-over-Push (Option):** While the vault uses Chainlink for automated push-distributions, it maintains internal accounting to prevent locked funds.
3. **Emergency Controls:** The Admin has the ability to manage roles in case of a security incident.

## 🐛 Bug Bounty
If you discover a security vulnerability, please do not open a public issue. Contact the developer directly to ensure a responsible disclosure process.
